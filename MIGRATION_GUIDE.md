## <a name="google-authenticator"></a>📘 Google Authenticator

### Шаг 1: Экспорт аккаунтов

1. Откройте **Google Authenticator** на телефоне
2. Нажмите **⋮** (три точки) в правом верхнем углу
3. Выберите **"Экспорт аккаунтов"**
4. Выберите аккаунты для экспорта (или все)
5. Нажмите **"Далее"**
6. Появится **QR-код** с данными экспорта

### Шаг 2: Получение данных экспорта

#### Вариант A: Через скриншот (рекомендуется)

1. Сделайте **скриншот** QR-кода
2. Откройте приложение для чтения QR (или веб-сканер)
3. Отсканируйте QR-код со скриншота
4. Скопируйте полученный текст (начинается с `otpauth-migration://`)

#### Вариант B: Через другой телефон

1. На втором телефоне откройте любое приложение для сканирования QR
2. Отсканируйте QR-код экспорта
3. Скопируйте полученный URI

### Шаг 3: Импорт в Immortal Web3 2FA

1. Откройте **Immortal Web3 2FA**
2. Нажмите **"📥 Import"**
3. Вставьте скопированный URI (`otpauth-migration://...`)
4. Нажмите **"Import"**
5. ✅ Аккаунты добавлены!

## <a name="authy"></a>🔵 Authy

### Шаг 1: Получение секретов из Authy

Authy не поддерживает прямой экспорт. Используйте один из методов:

#### Метод A: Через настройки каждого аккаунта

1. Откройте Authy
2. Для каждого аккаунта:
   - Нажмите на аккаунт
   - Выберите **"Edit"** или **"Settings"**
   - Найдите **"Secret Key"** или **"Show Secret"**
   - Скопируйте секрет

#### Метод B: Через резервную копию (Desktop)

1. Установите **Authy Desktop**
2. Войдите в аккаунт
3. Включите **Backups** в настройках
4. Файл резервной копии находится в:
   - **Windows**: `%APPDATA%\Authy\`
   - **Mac**: `~/Library/Application Support/Authy/`
   - **Linux**: `~/.config/Authy/`

5. Расшифруйте резервную копию (требуется пароль)
6. Извлеките секреты из JSON

#### Метод C: Через отладку Android (ADB)

```bash
# Требуется root доступ
adb pull /data/data/com.authy.authy/databases/authy.db
# Используйте SQLite браузер для открытия
# Таблица: tokens
```

### Шаг 2: Добавление в Immortal Web3 2FA

Для каждого извлеченного секрета:

1. Откройте **Immortal Web3 2FA**
2. Нажмите **"➕ Add"**
3. Введите:
   - **Service Name**: Название сервиса (например, "Gmail")
   - **Secret Key**: Секрет из Authy (Base32)
   - **Issuer**: (опционально) Название компании
4. Нажмите **"Add Service"**

## <a name="manual"></a>🔧 Ручное добавление сервисов

### Где найти Secret Key?

При настройке 2FA на большинстве сайтов:

1. Выберите **"Can't scan QR code?"** или **"Show secret key"**
2. Скопируйте отображаемый секрет (Base32 формат)
3. Вставьте в Immortal Web3 2FA

### Примеры популярных сервисов:

| Сервис | Где найти секрет |
|--------|------------------|
| **Google** | При настройке 2FA → "Показать ключ" |
| **GitHub** | Settings → Security → 2FA → Set up key → Enter manually |
| **Microsoft** | При настройке → "I want to set up a different authenticator app" |
| **Amazon** | Settings → Login & security → 2FA → Add → Manual entry |
| **Discord** | User Settings → My Account → Enable 2FA → Manual entry |

## <a name="qr-scan"></a>📷 Сканирование QR-кодов

### Прямое сканирование

1. В Immortal Web3 2FA нажмите **"📷 Scan QR"**
2. Разрешите доступ к камере
3. Наведите на QR-код 2FA
4. ✅ Сервис добавлен автоматически

### Поддерживаемые форматы:

- `otpauth://totp/SERVICE?secret=SECRET&issuer=ISSUER`
- `otpauth://hotp/SERVICE?secret=SECRET&counter=N`
- QR-коды с текстом содержащим Base32 секрет

## <a name="troubleshooting"></a>⚠️ Проблемы и решения

### ❌ "Invalid Base32 format"

**Причина**: Секрет содержит недопустимые символы

**Решение**:
- Удалите все пробелы
- Убедитесь что используются только символы `A-Z` и `2-7`
- Попробуйте привести к верхнему регистру

### ❌ "Service already exists"

**Причина**: Сервис с таким именем уже добавлен

**Решение**:
- Используйте уникальное имя (например, "Gmail Work" vs "Gmail Personal")
- Или удалите существующий сервис перед добавлением

### ❌ "Decryption failed. Wrong wallet?"

**Причина**: Подключен другой кошелек

**Решение**:
- Подключите тот же кошелек который использовался при шифровании
- Ключ шифрования генерируется из подписи кошелька

### ❌ QR-код не сканируется

**Причины**:
- Плохое освещение
- QR-код слишком маленький
- Камера не фокусируется

**Решения**:
- Увеличьте яркость экрана с QR-кодом
- Приблизьте/отдалите камеру
- Протрите объектив камеры
- Используйте скриншот QR-кода

### ❌ Коды не работают

**Причины**:
- Неправильное время на устройстве
- Неверный секрет

**Решения**:
1. Проверьте время на устройстве (должно быть точным)
2. В настройках 2FA сервиса выберите "Sync time"
3. Перепроверьте секрет

```
otpauth-migration://offline?data=BASE64_ENCODED_PROTOBUF_DATA
```

```protobuf

  message OtpToken {
    string name = 1;

  int32 version = 2;
  int32 batch_size = 3;
```
